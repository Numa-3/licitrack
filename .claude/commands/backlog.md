Lee `docs/backlog.md` y muestra un resumen del estado actual del backlog.

## Instrucciones

1. Lee el archivo `docs/backlog.md` completo.

2. Muestra el contenido organizado así:
   - Primero las **Mejoras pendientes** agrupadas por prioridad (alta → media → baja), con su número y módulo
   - Luego las **Features nuevas** agrupadas por prioridad, con nombre y una línea de qué hace
   - Al final una línea con el total: "X mejoras + Y features pendientes"

3. No incluyas la sección "Ya implementado" — solo lo pendiente.

4. Sé conciso — esto es una vista rápida, no un documento detallado.
