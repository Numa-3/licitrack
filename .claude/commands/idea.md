Guarda la siguiente idea en el backlog del proyecto: $ARGUMENTS

El archivo destino es `docs/backlog.md`.

## Instrucciones

1. Lee `docs/backlog.md` para entender la estructura actual.

2. Clasifica la idea según su tipo:
   - Si es un ajuste o mejora a algo que ya existe → sección "Mejoras a features existentes"
   - Si es una funcionalidad que no existe todavía → sección "Features nuevas"

3. Dentro de cada sección, ubícala en la subsección de prioridad correcta:
   - Si el usuario indica prioridad explícita (alta/media/baja) → usarla
   - Si no indica → inferirla del contexto (impacto en el flujo diario = alta, mejora de UX = media, nice-to-have = baja)

4. Para mejoras a features existentes, agrégala a la tabla correspondiente con:
   - Número correlativo (continuar desde el último)
   - Descripción concisa (máximo 10 palabras)
   - Módulo afectado

5. Para features nuevas, agrégala como una sección `####` con:
   - Nombre de la feature
   - Qué hace (1-2 líneas)
   - Por qué (qué problema resuelve)
   - Consideraciones técnicas si las mencionó el usuario

6. Confirma qué se agregó, en qué sección, y con qué prioridad.
